# Live Preview: требования к изображениям


## Главный принцип

Live Preview собирается из слоев:

1. подставка выбранного типа, материала и цвета;
2. дополнительные опции;
3. фигурки;
4. при необходимости отдельные передние/задние слои сложных опций.

У каждой подставки своя геометрия: отверстия под фигурки, зоны под блоки, ручку,
корзинку, стикеры и другие опции расположены по-разному. Поэтому live-preview не
должен использовать одну универсальную сетку для всех подставок.

Итоговый подход: **гибридный**.

- Подставки готовятся как полные canvas-слои.
- Фигурки и большинство опций готовятся как обрезанные cutout PNG.
- Координаты размещения фигурок и опций задаются отдельно для каждой подставки.
- Временные full-canvas слои фигурок/опций допустимы только для быстрой проверки
  визуала, но они не подходят для гибкой сборки всех комбинаций.

## Холст превью

- Формат сцены: `4:3`.
- Рекомендуемый размер итогового превью: `1600x1200 px`.
- Экспорт: прозрачный `PNG`.
- Все full-canvas слои должны быть строго `1600x1200 px`.
- Все слои должны быть сделаны в одной перспективе, с одним углом камеры и одной
  системой света/тени.

Если дизайнеру удобнее работать в `1000x750 px`, это допустимо только если **все**
full-canvas слои имеют один и тот же размер. Смешивать размеры нельзя.

## Два типа ассетов

### 1. Full-canvas слой

Это PNG размером `1600x1200`, где объект уже стоит в точном месте итоговой сцены.
Такой слой рендерится на всю сцену без дополнительных координат.

Используем для:

- подставок;
- вариантов цвета/материала подставки;
- временных мокапов для быстрой проверки;
- сложных случаев, где объект нельзя нормально собрать из cutout без отдельной
  передней/задней маски.

### 2. Cutout слой

Это PNG, обрезанный по видимому объекту, с прозрачным фоном и без лишнего пустого
пространства вокруг.

Используем для:

- матрешек;
- самовара;
- балалайки;
- ручек;
- блоков;
- еженедельника;
- закладок;
- стикеров;
- корзинки, если ее можно корректно поставить одним слоем.

Cutout-слой нужен, чтобы один и тот же объект можно было поставить в разные слоты
разных подставок.

## Подставки

У подставки один цвет равен одной картинке. Значит нужны отдельные PNG для каждой
пары:

```text
stand_code + material_variant_code
```

Формат:

```text
stand_1__acrylic_clear_3mm.png
stand_1__acrylic_red_3mm.png
stand_1__acrylic_blue_3mm.png
stand_1__acrylic_black_3mm.png
stand_1__acrylic_green_3mm.png
stand_1__acrylic_white_3mm.png
stand_1__plywood_default.png
```

То же самое нужно для `stand_2`, `stand_3`, `stand_4`, если эти материалы/цвета
доступны для конкретной подставки.

Требования:

- размер файла `1600x1200`;
- прозрачный фон;
- подставка должна быть в финальной перспективе;
- тень и свет должны совпадать между всеми цветами одной подставки;
- отверстия и зоны установки должны быть видны или понятны по эталонному мокапу.

## Layout по каждой подставке

Для каждой подставки нужна отдельная карта слотов. Это можно передать как схему,
таблицу или эталонный PSD/Figma-макет.

Минимально нужно указать:

- сколько фигурок допускает подставка;
- где стоят фигурки;
- какие размеры фигурок допустимы в каждом слоте;
- где располагаются блоки;
- где располагается ручка;
- где располагаются корзинка, стикеры, закладки и другие опции;
- какие опции могут перекрывать фигурки или подставку.

Пример структуры, которую разработка потом переносит в код:

```text
stand_2:
  figure_slots:
    left: большой/средний/малый слот
    center: большой/средний/малый слот
    right: большой/средний/малый слот
  option_slots:
    basket: слева, за/перед фигурками
    small_block: спереди справа
    medium_block: спереди справа
    weekly_notebook: спереди справа
    pens: по переднему краю
    stickers: зона справа/середина
    bookmarks: зона справа
```

Важно: `stand_1`, `stand_2`, `stand_3`, `stand_4` не обязаны иметь одинаковые
координаты. Наоборот, по умолчанию считаем, что у каждой подставки свои слоты.

## Фигурки

Нужны отдельные прозрачные cutout PNG для всех фигурок из логики:

```text
samovar.png
balalaika.png
matryoshka_khokhloma_big.png
matryoshka_khokhloma_medium.png
matryoshka_khokhloma_small.png
matryoshka_gorodets_big.png
matryoshka_gorodets_medium.png
matryoshka_gorodets_small.png
matryoshka_gzhel_big.png
matryoshka_gzhel_medium.png
matryoshka_gzhel_small.png
```

Требования:

- прозрачный фон;
- PNG обрезан по объекту, без холста `1600x1200` вокруг;
- нижняя точка объекта должна быть аккуратно выровнена, чтобы фигурка "стояла" на
  подставке;
- большой, средний и малый размеры должны быть пропорциональны реальным размерам;
- все фигурки должны быть в одной перспективе с подставками;
- не добавлять встроенную тень, если тень должна управляться слоем подставки или CSS.

Примечание: балалайка считается большой фигуркой отдельно от матрешек. Среди
матрешек больших видов три: Гжель, Городец/Городецкая, Хохлома. Вместе с самоваром
это четыре большие фигурки без учета балалайки.

## Опции

Нужны прозрачные cutout PNG:

```text
small_block.png
medium_block.png
weekly_notebook.png
pens.png
bookmarks.png
stickers.png
basket.png
```

Требования:

- PNG обрезан по видимому объекту;
- объект в той же перспективе, что и подставка;
- масштаб должен соответствовать реальному размеру относительно фигурок и подставки;
- для опций, которые лежат на подставке, нижний край/точка контакта должны быть
  понятны.

Для корзинки может понадобиться разделение на два слоя:

```text
basket_back.png
basket_front.png
```

Это нужно, если корзинка должна стоять частично за фигурками, но передняя сетка
должна перекрывать их. Сейчас frontend умеет работать с одним `basket.png`; разделение
на `front/back` можно подключить следующим шагом без изменения бизнес-логики.

## Эталонные мокапы

Чтобы быстро настроить координаты, нужны эталонные изображения для каждой подставки:

```text
stand_1_reference.png
stand_2_reference.png
stand_3_reference.png
stand_4_reference.png
```

На эталоне должны быть показаны:

- подставка;
- максимально типичная комбинация фигурок;
- основные доступные опции;
- пример того, как должны выглядеть пропорции.

Эти файлы не обязательно подключаются в сайт. Они нужны разработке как визуальная
линейка для настройки layout-координат.

## Что можно прислать первым пулом

Минимальный набор для быстрой калибровки:

```text
stand_2__acrylic_clear_3mm.png
stand_2__acrylic_red_3mm.png
stand_2__acrylic_blue_3mm.png
stand_2__plywood_default.png
matryoshka_khokhloma_big.png
matryoshka_gorodets_big.png
matryoshka_gzhel_big.png
small_block.png
medium_block.png
weekly_notebook.png
pens.png
basket.png
stand_2_reference.png
```

После этого можно быстро настроить одну подставку, проверить масштаб и затем
размножить подход на остальные подставки.

В текущем пуле cutout-ассетов `weekly_notebook.png` еще отсутствует. До его появления
старый full-canvas preview еженедельника не подключаем к slot-превью.

## Изображения для карточек товаров

Карточные изображения - это отдельные ассеты для каталога, главной страницы,
корзины и шагов конфигуратора. Их не нужно готовить как live-preview слои.
Live-preview слой отвечает за сборку сцены, а карточная картинка отвечает за
быстрое визуальное узнавание товара.

Главное правило: **не использовать full-canvas preview `1600x1200` как картинку
карточки товара**. На карточке такой ассет выглядит слишком мелким, потому что
внутри него много пустого пространства сцены.

### Общие требования

- Форматы: `JPG`/`WEBP` для фотографий, `PNG` для прозрачных вырезок.
- Цветовой профиль: `sRGB`.
- Фон: прозрачный или светлый нейтральный. Черный фон допустим только если он
  является частью утвержденного стиля, но не должен занимать большую пустую
  площадь вокруг объекта.
- Без текстовых плашек, надписей размеров, водяных знаков, служебных меток и
  подписей вроде "высокая", "средняя", "малая".
- Объект должен занимать большую часть кадра: обычно `70-90%` полезной площади.
- Без лишнего пустого поля вокруг объекта. Безопасные поля по краям: примерно
  `5-8%`.
- Все изображения одной группы должны быть в одной перспективе и с похожим
  светом, чтобы сетка карточек не выглядела случайной.
- Названия файлов - только по стабильным `code`, латиницей, без русских
  названий товаров и пробелов.

### Фото направлений и готовых наборов

Используются на главной странице в карточках направлений бизнеса и блоках
готовых наборов.

- Рекомендуемая пропорция: `16:10`.
- Рекомендуемый размер исходника: `1600x1000 px` или `1400x875 px`.
- Набор должен быть крупным и читаемым, без больших пустых зон вокруг.
- Кадр должен выдерживать легкий `cover`-кроп: важные детали не должны стоять
  вплотную к краям.
- Если исходная студийная фотография на черном фоне, лучше подготовить отдельный
  card-crop: ближе к объекту, меньше черной пустоты, больше самого набора.
- Для карточки направления не добавлять рекламный текст поверх изображения.
  Название и описание уже выводятся интерфейсом.

### Подставки

Используются в шаге выбора подставки в конфигураторе.

- Рекомендуемая пропорция: `4:3`.
- Рекомендуемый размер: `1200x900 px` или `1600x1200 px`.
- Подставка должна занимать `80-90%` ширины кадра.
- Отверстия, держатели, зоны под блоки и ручки должны быть хорошо видны.
- Перспектива должна быть такой же или очень близкой к live-preview, чтобы
  пользователь узнавал выбранную подставку в превью.
- Для вариантов материала можно использовать один и тот же ракурс, меняя только
  цвет/материал.

### Фигурки

Используются в шаге выбора фигурок.

- Рекомендуемая пропорция: портретная `3:4` или близкая к ней.
- Рекомендуемый размер: `900x1200 px`, `1000x1200 px` или больше в той же
  пропорции.
- Фигурка должна занимать `75-90%` высоты кадра.
- Нижняя точка фигурки должна быть выровнена одинаково для всей серии, чтобы
  большие, средние и малые карточки смотрелись аккуратно.
- Самовар и балалайка должны быть подготовлены как отдельные чистые карточные
  изображения, без подписи размера в самом изображении.
- Для маленьких и средних матрешек не нужно показывать реальный масштаб слишком
  буквально. Карточка должна быть наглядной: пользователь должен видеть рисунок
  и форму, а реальный размер уже указан текстом рядом.
- Если используется прозрачный PNG, объект должен быть обрезан близко к силуэту,
  без большого невидимого холста вокруг.

### Опции

Используются в шаге выбора дополнительных элементов.

- Рекомендуемая пропорция: `4:3` для объемных объектов, `16:10` для длинных
  объектов вроде ручек, закладок и стикеров.
- Рекомендуемый размер: минимум `1000x750 px`.
- Объект должен занимать `70-85%` кадра.
- Длинные тонкие предметы лучше располагать слегка по диагонали или группой,
  чтобы они не превращались в тонкую линию на карточке.
- Для блоков, еженедельников и корзинок важно показать форму и контактную
  поверхность, чтобы было понятно, как объект встанет на подставку.

### Мини-чеклист перед загрузкой

- На карточке объект виден крупно без приближения.
- Нет встроенного текста, плашек, водяных знаков и технических подписей.
- Нет лишнего черного или прозрачного поля вокруг объекта.
- Изображение выглядит хорошо в маленькой карточке шириной около `200-360 px`.
- Серия товаров выглядит единообразно: одинаковый свет, ракурс, фон и плотность
  кадра.

## Что нельзя делать

- Нельзя смешивать разные размеры холста для full-canvas слоев.
- Нельзя отдавать фигурки и опции как full-canvas слои, если они должны свободно
  переставляться между слотами.
- Нельзя оставлять много пустого пространства вокруг cutout-объекта.
- Нельзя называть файлы русскими названиями товаров. Нужны стабильные `code`.
- Нельзя смешивать вид сверху, фронтальный вид и перспективу в одном наборе.
- Нельзя зашивать один и тот же объект в разные позиции без понятной причины: это
  быстро ломает динамическую сборку.

## Текущее состояние frontend

Frontend сейчас поддерживает оба режима:

- `canvas` - слой растягивается на всю сцену `4:3`;
- `slot` - cutout-слой ставится в координаты конкретной подставки.

Временный первый пул full-canvas фигурок и опций может отображаться нормально в
`canvas`-режиме. Для полноценного конфигуратора и всех комбинаций целевой формат -
cutout-фигурки, cutout-опции и отдельные layout-слоты для каждой подставки.
